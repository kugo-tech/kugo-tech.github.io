Stack

Our Tools

Go Microservices

When exponential demand is anticipated we usually plan the solution as a set of Go microservices deployed to Kubernetes. This ensures us pain-free scalability and absolute portability. GKE, EKS & DigitalOcean Kubernetes are our most used cloud providers but nothing prevents moving our implementation to any other service that supports this technology.

Clojure

If the scope of the application is widely established and high attendance with uniform demand is expected we prefer to develop them using Clojure, it is really powerful when it comes to data manipulating and also excellent at concurrency, parallelism is something very easy to achieve. Clojure runs on the batllee tested JVM, interoperation with its huge ecosystem of Java libraries. Finally, Clojure let us express a lot with very little code, this ends up being vital when development speed is almost mandatory to achieve a fast time-to-market solution.

ClojureScript

When more complex user interactions are required and server side rendering falls short for the kind of user experience required, our primary choice to develop Single Page Applications is ClojureScript & re-frame as a tandem. This functional language along with its immutable data type conform the best-in-class frontend development environment that one can aspire to: Instant hot-code reloading, a browser-connected REPL, and the biggest collection of libraries for getting stuff done on time. Integrating it with React Native conforms an easy path to later move the app to Android or iOS.
